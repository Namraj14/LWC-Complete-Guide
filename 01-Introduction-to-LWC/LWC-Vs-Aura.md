# What is LWC?

**Lightning Web Components (LWC)** is Salesforce's modern UI framework used to build fast, reusable, and scalable web components on the Salesforce Platform.

LWC is built on **native web standards**, which means it uses the same technologies that modern websites use.

It is based on:

- HTML
- CSS
- JavaScript (ES6+)
- Web Components
- Shadow DOM
- Custom Elements
- ECMAScript Modules

Unlike Aura Components, LWC doesn't try to recreate browser functionality. Instead, it uses the browser's built-in capabilities.

---

# Why Salesforce Created LWC

Before LWC, Salesforce developers used **Aura Components** to build Lightning applications.

Aura was a significant improvement over Visualforce because it introduced:

- Component-based architecture
- Event-driven programming
- Dynamic user interfaces
- Component reusability

However, as web technologies evolved, Aura started showing several limitations.

Salesforce wanted a framework that was:

- Faster
- Smaller
- Easier to maintain
- Based on industry standards
- Future-proof

This led to the introduction of **Lightning Web Components (LWC)** in **2019**.

---

# Problems with Aura Components

## 1. Heavy Framework

Aura depends on its own JavaScript framework.

Whenever a component loads, the browser first loads the Aura engine before rendering your component.

```
Browser
      ↓
Aura Framework
      ↓
Aura Component
      ↓
HTML
```

This additional framework layer increases loading time.

---

## 2. Slower Performance

Every interaction passes through the Aura framework.

Example:

```
Button Click
      ↓
Aura Framework
      ↓
Controller
      ↓
Component Update
```

Extra processing results in slower rendering compared to native browser rendering.

---

## 3. Proprietary Framework

Aura is Salesforce-specific.

Developers must learn Aura syntax that cannot be reused outside Salesforce.

Example:

```xml
<aura:component>
```

This syntax is unique to Salesforce.

---

## 4. Difficult Debugging

Errors generated by Aura can be difficult to understand.

Example:

```
Action failed:
c:AccountComponent$controller$save
```

Since Aura handles much of the rendering internally, browser developer tools are less effective.

---

## 5. Doesn't Fully Use Modern Browser Features

Modern browsers already support:

- JavaScript Classes
- Modules
- Shadow DOM
- Custom Elements

Aura recreates many of these features instead of using them directly.

---

# Salesforce's Solution

Instead of building another custom framework, Salesforce chose to build on top of native browser standards.

Lightning Web Components directly use:

- HTML
- CSS
- JavaScript
- Web Components
- ES6 Classes
- Shadow DOM
- Browser Rendering Engine

Result:

- Better performance
- Smaller framework
- Easier development
- Better debugging
- Industry-standard skills

# Browser Architecture

Every web page goes through several processing stages before it appears on the screen.

```
Salesforce Server
        │
        ▼
Browser receives
        │
        ├── HTML
        ├── CSS
        └── JavaScript
```

The browser then processes these files.

---

# Step 1 — HTML Parsing

The browser reads HTML and creates the **Document Object Model (DOM)**.

Example:

```html
<body>
    <h1>Hello</h1>
    <button>Save</button>
</body>
```

DOM Tree:

```
Document
└── html
    └── body
        ├── h1
        └── button
```

The DOM represents every HTML element as an object.

---

# Step 2 — CSS Parsing

The browser reads CSS and creates the **CSS Object Model (CSSOM)**.

Example:

```css
h1 {
    color: blue;
}
```

The CSSOM stores all styling information.

---

# Step 3 — Render Tree

The browser combines the DOM and CSSOM.

```
DOM
 +
CSSOM
  ↓
Render Tree
```

Only visible elements become part of the Render Tree.

---

# Step 4 — Layout (Reflow)

The browser calculates:

- Element size
- Width
- Height
- Margins
- Position
- Spacing

This determines where every element will appear.

---

# Step 5 — Paint

The browser draws everything onto the screen.

This is the final stage where users see the webpage.

---

# JavaScript and the DOM

JavaScript can modify the DOM dynamically.

Example:

```javascript
document.querySelector('h1').textContent = 'Welcome';
```

The browser updates the DOM and repaints only the affected portion of the page.

This allows web applications to update content without refreshing the entire page.

---

# Why Browser Architecture Matters in LWC

LWC relies on the browser's native rendering engine instead of implementing its own.

```
LWC Component
      │
      ▼
Native JavaScript
      │
      ▼
Browser DOM
      │
      ▼
Layout
      │
      ▼
Paint
```

Because of this architecture:

- Components render faster.
- Less JavaScript is executed.
- Memory consumption is lower.
- UI updates are more efficient.
- Applications feel more responsive.

# Interview Questions

### Why did Salesforce introduce LWC?

Salesforce introduced Lightning Web Components to improve performance, reduce framework overhead, leverage native browser capabilities, follow modern web standards, and provide a faster, more maintainable component framework than Aura.

---

### Why is LWC faster than Aura?

Because LWC relies on native browser APIs instead of routing rendering and events through a custom JavaScript framework.

---

### What technologies are used in LWC?

- HTML
- CSS
- JavaScript
- Web Components
- Shadow DOM
- ES6 Modules
- Custom Elements

---

### What is the DOM?

The Document Object Model (DOM) is a tree-like representation of an HTML document that allows JavaScript to read and modify web page elements dynamically.

---

### What is the Render Tree?

The Render Tree is created by combining the DOM and CSSOM. It contains only the visible elements that the browser will display.

---

### What are the stages of browser rendering?

1. HTML Parsing
2. DOM Creation
3. CSS Parsing
4. CSSOM Creation
5. Render Tree Creation
6. Layout (Reflow)
7. Paint

---

# Key Takeaways

- LWC is Salesforce's modern component framework.
- LWC is built using native web standards.
- Aura depends heavily on its own framework, while LWC leverages the browser directly.
- LWC provides better performance, easier debugging, and a simpler development model.
- Understanding browser architecture is essential because LWC relies on the browser's rendering engine.
- The browser renders pages by creating the DOM, CSSOM, Render Tree, performing Layout, and finally Painting the UI.
